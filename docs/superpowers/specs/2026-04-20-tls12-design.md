# TLS 1.2 支持设计文档

**日期：** 2026-04-20  
**状态：** 待实现

## 背景

当前 TLS 实现仅支持 TLS 1.3。部分服务器（如 baidu.com）返回 `protocol_version` alert（desc=70），
原因是服务器选择 TLS 1.2，而客户端不支持该路径。需要添加 TLS 1.2 支持并实现自动版本协商。

## 目标

- 支持 TLS 1.2（`TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256`、`TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256`）
- 单次 ClientHello 协商：同时宣告 TLS 1.3 + TLS 1.2，服务器选择版本，客户端分叉处理
- 零新依赖，全部复用现有加密原语
- 将现有 TLS 1.3 代码整理进 `tls13/`，与新 `tls12/` 对称

## 握手流程（TLS 1.2 路径）

```
Client                                  Server
  |-- ClientHello (supported_versions: 1.3, 1.2) ---> |
  |<-- ServerHello (selected: 1.2) ------------------- |
  |<-- Certificate ------------------------------------ |
  |<-- ServerKeyExchange (P-256 公钥 + 签名) ---------- |
  |<-- ServerHelloDone -------------------------------- |
  |--- ClientKeyExchange (P-256 公钥) ----------------> |
  |--- ChangeCipherSpec ------------------------------> |
  |--- Finished (加密) --------------------------------> |
  |<-- ChangeCipherSpec ------------------------------- |
  |<-- Finished (加密) -------------------------------- |
  |=== 应用数据 ======================================= |
```

分叉点：解析 ServerHello 后检查 `server_version`：
- `0x0303` → 进入 TLS 1.2 握手路径（`Tls12Handshake`）
- `0x0304` → 保持现有 TLS 1.3 路径（`Tls13Handshake`）

## 文件结构

```
src/core/net/tls/
  tls12/
    mod.rs          # Tls12Handshake 状态机（新增）
    key_schedule.rs # PRF、master_secret、key_block 推导（新增）
    record.rs       # TLS 1.2 记录加密/解密（显式 IV）（新增）
  tls13/
    mod.rs          # Tls13Handshake（从 handshake.rs 移入并重命名）
    key_schedule.rs # 从 key_schedule.rs 移入
    record.rs       # 从 record.rs 移入
  codec.rs          # 共用编解码原语（扩展：ClientKeyExchange、ServerKeyExchange）
  mod.rs            # TlsStream / AsyncTlsStream，connect 负责版本分叉
  crypto/           # 不动
  p256.rs / rsa.rs / ecdsa.rs / x509.rs / ecdsa.rs  # 不动
```

## 密码套件

| 套件 | ID | 密钥交换 | 加密 | MAC |
|------|----|---------|------|-----|
| TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256 | 0xC02B | P-256 ECDHE | AES-128-GCM | SHA-256 |
| TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256 | 0xC02C | P-256 ECDHE | AES-128-GCM | SHA-256 |

两个套件均复用现有原语（`p256.rs`、`crypto/aes_gcm.rs`、`crypto/sha256.rs`）。

## 关键新增组件

### tls12/key_schedule.rs

```rust
// P_hash: HMAC-SHA256 数据扩展函数（RFC 5246 Section 5）
fn p_hash(secret: &[u8], seed: &[u8], out: &mut [u8])

// PRF = P_SHA256(secret, label || seed)
pub fn prf(secret: &[u8], label: &[u8], seed: &[u8], out: &mut [u8])

// master_secret = PRF(pre_master, "master secret", client_random + server_random)
pub fn master_secret(pre_master: &[u8], client_random: &[u8; 32], server_random: &[u8; 32]) -> [u8; 48]

// key_block 推导（write_key、write_iv × 2）
pub struct KeyBlock {
    pub client_write_key: [u8; 16],
    pub server_write_key: [u8; 16],
    pub client_write_iv:  [u8; 4],   // implicit nonce
    pub server_write_iv:  [u8; 4],
}
pub fn key_block(master: &[u8; 48], client_random: &[u8; 32], server_random: &[u8; 32]) -> KeyBlock

// Finished 验证数据（12 字节）
pub fn finished_data(master: &[u8; 48], label: &[u8], transcript_hash: &[u8; 32]) -> [u8; 12]
```

### tls12/record.rs

TLS 1.2 AES-GCM 记录与 TLS 1.3 的关键差异：

| 项目 | TLS 1.3 | TLS 1.2 |
|------|---------|---------|
| Nonce | 全隐式（seq XOR iv） | 4B 隐式 + 8B 显式（附在密文前） |
| 内容类型 | 明文中附加内部类型字节 | 无内部类型字节 |
| 记录版本字段 | `0x0303`（兼容） | `0x0303` |
| AAD | type+legacy_version+length | type+version+length+seq |

### tls12/mod.rs（Tls12Handshake 状态机）

```
状态：
  ExpectCertificate
  ExpectServerKeyExchange
  ExpectServerHelloDone
  [发送 ClientKeyExchange + CCS + Finished]
  ExpectChangeCipherSpec
  ExpectFinished
  Complete
```

`handle_message` 接收从外部已解析的握手消息类型，逐步推进状态。
`Tls12Handshake` 持有：P-256 临时私钥、client/server random、transcript SHA-256、已推导的 key_block。

### codec.rs 扩展

新增函数：
- `encode_client_hello_v12_extension`：在现有 `encode_client_hello` 中追加 TLS 1.2 密码套件，`supported_versions` 同时列出 `0x0304` 和 `0x0303`
- `encode_client_key_exchange(public_key: &[u8; 65]) -> Vec<u8>`：发送 P-256 非压缩公钥
- `parse_server_key_exchange(data: &[u8]) -> Result<(curve_info, pubkey, signature)>`：解析 ECDHE 参数

## 不在范围内

- TLS 1.1 及以下
- TLS 1.2 CBC 模式（只做 GCM）
- Session resumption / Session tickets
- Client certificate authentication
- TLS 1.2 async 路径（`AsyncTlsStream`，后续按需添加）
