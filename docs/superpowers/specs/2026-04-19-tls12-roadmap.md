# TLS 1.2 Client: 精简生产级实现路线图

## 背景与动机

viv 当前的 TLS 客户端（`src/core/net/tls/`）只实现 TLS 1.3，使用 x25519 + AES-GCM + SHA256，~2800 LOC。但实际联网时存在覆盖缺口：

- `www.baidu.com` 等国内主流站点只协商 TLS 1.2 → 我们收到 `protocol_version` 告警（desc=70），无法建连
- 部分 API/CDN 为兼容性保留 TLS 1.2 → 与这些端点通信时必须回退

本文档定义一个**精简生产级 TLS 1.2 客户端**的分阶段实现路线图，目标是"HTTPS 能访问市面上大多数网站"。每个阶段是独立的子项目，各自走 brainstorm → spec → plan → 实现循环。

## 目标与非目标

**目标**
- 与 TLS 1.3 客户端共存，通过 ClientHello 的 `supported_versions` 扩展双版本协商
- 覆盖 RSA + ECDSA 两类证书（现代 HTTPS 流量的 95%+）
- 完整的证书链验证（包含主机名 SAN 匹配、有效期、签名链到根 CA）
- 零依赖、跨平台、TDD

**非目标（本次不做）**
- ChaCha20-Poly1305 AEAD（AES-GCM 已覆盖）
- 会话恢复（Session Ticket / Session ID）——性能优化，非功能必需
- mTLS（客户端证书）——企业场景，公网访问用不到
- ALPN / HTTP/2 协商——只默认走 HTTP/1.1，不宣告 ALPN 扩展
- 强制 RSA 密钥交换（非 ECDHE）——这类老服务器已退市
- 客户端证书选择/revocation / OCSP / CT log 验证——精简范围外

## 支持的 Cipher Suites

严格限定为四个（覆盖 95%+ 现代 HTTPS）：

- `TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256` (0xC02F)
- `TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384` (0xC030)
- `TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256` (0xC02B)
- `TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384` (0xC02C)

## Phase 拆分

| # | 名称 | LOC | 依赖 | 目的 |
|---|---|---:|---|---|
| 1 | **BigUint** | ~500 | — | 加减乘除、模幂、模逆，RSA 运算基础 |
| 2 | **ASN.1/DER 解析器** | ~400 | — | 支持 X.509/PKCS#8 的语法层 |
| 3 | **RSA PKCS#1 v1.5 签名验证** | ~250 | 1 | 验证 RSA 签名（证书签名、ServerKeyExchange 签名） |
| 4 | **X.509 证书解析** | ~500 | 2 | 提取 subject/issuer/SAN/validity/公钥/签名算法 |
| 5 | **P-256 + ECDSA 签名验证** | ~600 | 1 | secp256r1 曲线 + ECDSA 签名验证 |
| 6 | **证书链验证 + 精简 Mozilla 根 CA** | ~500 | 3, 4 | 建链 + 主机名（SAN）匹配 + 有效期 + 签名链验证 |
| 7 | **TLS 1.2 核心协议** | ~800 | 1–6 | 握手状态机 + 记录层 + 双版本协商 |

**总计：~3550 LOC**

## Phase 依赖图

```
      Phase 1 (BigUint)  Phase 2 (ASN.1)
           │  │              │
           │  └─ Phase 3 (RSA verify)
           │                 │
           └─ Phase 5 (ECDSA)│
                        │    └── Phase 4 (X.509)
                        │            │
                        └────────────┴──── Phase 6 (Chain + Roots)
                                              │
                                Phase 7 (TLS 1.2 Protocol)
```

- **可并行**：Phase 1 与 Phase 2（互不依赖）
- **关键路径**：1 → 3 → 6 → 7；2 → 4 → 6 → 7
- **Phase 7 前提**：Phase 1–6 全部落地

## 验收场景（Phase 7 完成后）

- ✅ `www.baidu.com` HTTPS GET：RSA 证书 + TLS 1.2
- ✅ `www.cloudflare.com` HTTPS GET：ECDSA 证书 + TLS 1.3（已支持）回退 1.2
- ✅ `api.anthropic.com` HTTPS GET：RSA + TLS 1.3（已支持）
- ✅ 任意 Let's Encrypt 签发站点
- ✅ 过期证书：拒绝连接并报 `Error::Tls("certificate expired")`
- ✅ SAN 主机名不匹配：拒绝连接
- ✅ 自签证书：默认拒绝（除非链到受信根）

## 目录约定

`src/core/net/tls/` 下按 Phase 扩展：

```
src/core/net/tls/
├── codec.rs            # (现有) TLS record / handshake 编解码
├── handshake.rs        # (现有) TLS 1.3 握手状态机
├── key_schedule.rs     # (现有) TLS 1.3 key schedule
├── record.rs           # (现有) TLS record layer
├── mod.rs              # (现有) TlsStream / AsyncTlsStream
├── bigint.rs           # [Phase 1] BigUint
├── asn1.rs             # [Phase 2] ASN.1/DER parser
├── rsa.rs              # [Phase 3] RSA PKCS#1 v1.5 verify
├── x509.rs             # [Phase 4] (替换现有空壳) X.509 证书解析
├── ecdsa.rs            # [Phase 5] P-256 + ECDSA verify
├── verify.rs           # [Phase 6] 证书链验证 + 主机名匹配
├── roots.rs            # [Phase 6] 打包的 Mozilla 根 CA (DER 二进制数组)
├── handshake12.rs      # [Phase 7] TLS 1.2 握手状态机
├── key_schedule12.rs   # [Phase 7] TLS 1.2 PRF-based key derivation
└── crypto/             # (现有)
    ├── aes_gcm.rs
    ├── sha256.rs
    ├── x25519.rs
    ├── hmac.rs         # [Phase 7] HMAC-SHA256 (PRF 需要)
    ├── sha384.rs       # [Phase 7] SHA-384 (AES_256_GCM_SHA384 需要)
    └── p256.rs         # [Phase 5] P-256 曲线算术
```

## 项目级约束

- **零依赖**：不引入任何 `[dependencies]`
- **跨平台**：`#[cfg]` 隔离 Linux/macOS/Windows 差异
- **TDD**：每个 phase 先写测试（`tests/core/net/tls/`）再实现
- **禁止崩溃代码**：`src/` 内不允许 `unwrap/expect/panic/unreachable/todo`
- **测试超时**：每个 `#[test]` ≤10s；网络测试（已有 `full_test` feature）显式配超时
- **每个 phase 独立 commit 链**：不合并到一个巨型 PR
- **每个 phase 独立 spec**：`docs/superpowers/specs/2026-04-XX-tls12-phase<N>-<topic>-design.md`

## 执行节奏

1. 本 roadmap 提交后，进入 **Phase 1**（BigUint）详细 brainstorm
2. Phase 1 完成（spec + 实现 + 测试 + commit），回到 roadmap，启动 Phase 2
3. 依此推进到 Phase 7
4. Phase 7 完成后，启用 `tests/core/net/tls/baidu_test.rs` 中的 baidu 测试（目前为 `#[ignore]`）
